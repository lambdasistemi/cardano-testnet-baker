## ready

Round 7 of T002 — addresses the round-6 verdict on `25b36b2`.

The reviewer's three correctness points were valid and aligned:

1. **`spec.md` FR-006** still promised "byte-identical artifact
   manifest digest" while the check excludes `history`. The OCI
   manifest digest includes the config digest which includes
   `history`, so the wording was internally inconsistent — the
   gate could pass while the actual manifest digest differs.
2. **`publish-pipeline.md`** said "the property the consumer
   pulling by manifest digest actually depends on" alongside
   layer-payload equality. A consumer pulling by manifest digest
   depends on the *full* manifest digest, not just `layer.tar`.
3. **`nix/checks.nix` comment** still claimed manifest-digest
   equality across the test pair, but the check intentionally
   strips `history`.

Round 7 retracts the manifest-digest framing throughout and
names the property the gate actually enforces:

- **FR-006** (spec.md) is rewritten to require:
  (a) byte-identical `layer.tar` bytes and
  (b) byte-identical image-config fields modulo `history`,
  and explicitly stops short of manifest-digest equality. The
  paragraph below explains *why* the test-pair manifest digest
  cannot match (dockerTools' `history.comment` store-path
  embedding) and notes that production CI's single-build path
  still pushes a deterministic manifest digest because the
  customisation layer's store path is fully determined by source
  + flake lock.
- **publish-pipeline.md §"Determinism check"** is rewritten to
  describe the two distinct properties the gate verifies
  ("Deterministic seed payload" and "Stable image-config
  fields"), drops the inaccurate "consumer pulling by manifest
  digest" claim, and renames the rationale section to "Why the
  gate does not compare the manifest digest". The
  production-vs-test-pair distinction is spelled out.
- **`nix/checks.nix`** comment is rewritten to match: the
  check oracle is "seed payload equality + stable
  consumer-facing config fields", and it explicitly notes that
  the manifest digest is *not* compared across the test pair.
