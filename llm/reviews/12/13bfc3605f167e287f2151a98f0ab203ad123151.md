## ready

T003 from `specs/003-seed-distribution/tasks.md` —
`compose/acceptance/run.sh` accepts `docker-archive:<path>` and
`oci-archive:<path>` URIs in addition to the existing directory
form. Lands on top of T002 (`d16b526`).

### What

- `compose/acceptance/run.sh`:
  - Adds `resolve_input_kind` and `extract_archive_seed`
    helpers. The latter:
    1. Allocates a tmpfs-backed mktemp dir
       (`mktemp -d -p /dev/shm ...`) with a fallback to
       `${TMPDIR:-/tmp}` if `/dev/shm` is unwritable (matches
       the contract clause in
       [`publish-pipeline.md §"Steps"`](../specs/003-seed-distribution/contracts/publish-pipeline.md));
    2. Runs `skopeo copy "" "dir:"` to materialise
       the OCI archive in directory form;
    3. Parses the resulting `manifest.json` with `jq` to
       locate the *single* layer blob (the seed image carries
       exactly one layer per the layout contract); rejects
       archives with anything other than one layer;
    4. Extracts that layer with `tar` into `<archive_extract_dir>/seed-root/`
       and returns `<seed-root>/seed` as the `baked_output_dir`
       the existing startup probe consumes.
  - Cleanup of the archive extraction dir is wired into the
    same `trap cleanup EXIT` that already removes
    `tmp_root`. Honours `ACCEPTANCE_KEEP_TMP=true` for the
    archive dir as well, for diagnostic runs.
  - The dir-mode path (current CI compose-acceptance flow)
    is unchanged — same `cp -R "/." "/"`,
    same topology + system-start patching.

## changes-requested

- `compose/acceptance/run.sh:58` / `compose/acceptance/run.sh:113`:
  archive extraction cleanup is not actually wired into the parent
  shell. `extract_archive_seed` assigns the global
  `archive_extract_dir`, but the archive branch calls it via command
  substitution (`baked_output_dir=$(extract_archive_seed "$input")`).
  Bash runs that function in a subshell for the command substitution, so
  the parent process still has `archive_extract_dir=` when the `trap`
  fires. The archive input path therefore leaves the mktemp extraction
  tree under `/dev/shm` or `$TMPDIR`, violating T003's acceptance item
  that archive mode leaves no residue and that cleanup happens via the
  trap. Please avoid command substitution for the function that mutates
  cleanup state, or allocate/export the archive temp dir in the parent
  shell before extracting.
