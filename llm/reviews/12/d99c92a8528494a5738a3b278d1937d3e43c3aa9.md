## ready

T001 from `specs/003-seed-distribution/tasks.md` — first
implementation patch of the speckit task list, gated by the now-
approved dep-bump (`7b32aec`).

### What

- `nix/seed-image.nix` (new): exports `mkSeedImage { scenarioName,
  scenarioPath }`. Bakes the scenario via `pkgs.runCommand`,
  assembles a `/seed/` tree by copying every bake artifact
  byte-for-byte except `synthesis-report.json`, which is replaced
  by `jq 'del(.observation)' synthesis-report.json` (Feature 002's
  determinism rule, research §8). Wraps the assembled tree in
  `pkgs.dockerTools.buildLayeredImage` with
  `created = "1970-01-01T00:00:00Z"`,
  `architecture = "amd64"`,
  `name = "ghcr.io/lambdasistemi/cardano-testnet-seed"`,
  no entrypoint, empty `config = { }`. Uses `contents = [ seedTree
  ]` (the parameter this nixpkgs vintage exposes; `copyToRoot` is
  not yet available here — the customisation layer the streamer
  emits already symlinks `/seed/` into the image root, so consumers
  see `/seed/` regardless).

- `flake.nix`: imports `nix/seed-image.nix`, enumerates
  `examples/scenarios/*.json` via `builtins.readDir` and
  `pkgs.lib.filterAttrs`, and merges `seedImage-<scenario>` into
  `packages.<system>` for every committed scenario. No manual
  scenario list anywhere in the flake.

### Contract reconciliation (docs travel with code)

The contract `seed-image-layout.md` and `data-model.md` previously
listed file names that diverged from the baker's actual output —
`alonzo.json` instead of `alonzo-genesis.json`, `op.cert` instead
of `opcert.cert`, `faucet-N/utxo.skey` instead of
`utxo-keys/genesis.<n>.skey`. Antithesis's
`/code/cardano-node-antithesis/components/configurator/configurator.sh`
expects the baker-shaped names (`byron-genesis.json`,
`opcert.cert`, `genesis.1.skey`), and so does cardano-node itself
(`--shelley-operational-certificate opcert.cert`,
`ByronGenesisFile = "byron-genesis.json"` in `config.json`). The
contract is updated to reflect what the baker actually emits;
T001 then ships a byte-for-byte copy. The file-mode contract is
also relaxed from the aspirational `0644`/`0755` to `0444`/`0555`
because Nix-store paths are read-only by design — `0644` would
require manual chmod inside the customisation layer, which only
helps for the symlinks, not the actual files the symlinks point
at.
