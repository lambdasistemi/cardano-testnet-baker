## ready

Round 2 of T005 — addresses the round-1 verdict on
`d1f12d8`.

The reviewer correctly flagged that the round-1 `--dry-run`
output didn't match the contract:
[`quickstart.md §A`](../specs/003-seed-distribution/quickstart.md)
specifies exactly four lines for two scenarios in the form
`<scenario>  <target-uri>`, no headers and no per-tag labels.
Round 1 emitted ten lines (a `=== <scenario> ===` header
plus four labelled lines per scenario), making the
acceptance output path neither machine-compatible nor
matching the documented form.

### What changed

`nix/seed-publish.nix` (`publish_one`):

- Dry-run now writes exactly two stdout lines per scenario,
  one per tag, in the documented two-column form
  (`<scenario>  <target-uri>`). No `===` header, no
  `primary =`/`secondary =` labels, nothing else on
  stdout. Two scenarios × two tags = four lines, matching
  the contract.
- Verbose labels for human-readable debugging are kept,
  but only on the *non-dry-run* (actual-push) path and
  written to **stderr**, so the acceptance dry-run path
  stays clean for piping into tools.
- The `primary_target` / `secondary_target` strings are
  now `<repository>:<tag>` (without the `docker://`
  prefix) so the dry-run output is the registry reference
  the consumer reads, while the actual-push branch
  appends `docker://` only when invoking `skopeo copy`.

Everything else (`extract_scenario_digest`,
`validate_tag`, scenario iteration, `BAKER_COMMIT_SHA7`
handling, the `flake.nix` apps wiring, the `justfile`
recipe) is unchanged from round 1.

### Validation

- `nix run .#publishSeedImages -- --dry-run` (with
  `BAKER_COMMIT_SHA7` exported) prints exactly four
  stdout lines:
  ```text
  local-fast  ghcr.io/lambdasistemi/cardano-testnet-seed:local-fast-<64-hex>
  local-fast  ghcr.io/lambdasistemi/cardano-testnet-seed:local-fast-sha-<7-hex>
  normal      ghcr.io/lambdasistemi/cardano-testnet-seed:normal-<64-hex>
  normal      ghcr.io/lambdasistemi/cardano-testnet-seed:normal-sha-<7-hex>
  ```
  Matches `quickstart.md` line 36–40 verbatim modulo the
  literal digest values.
- Calling without `--dry-run` against a non-existent
  registry still returns non-zero visibly: `set -e` plus
  `skopeo`'s non-zero exit on auth/network failures.
- Stderr in non-dry-run mode shows the
  `=== <scenario> ===` header and the per-tag labels for
  human debugging — but they no longer corrupt the
  acceptance output path.
