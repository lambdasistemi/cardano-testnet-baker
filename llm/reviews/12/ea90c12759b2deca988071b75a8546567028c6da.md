  `dockerTools.buildLayeredImage` with
  `created = "1970-01-01T00:00:00Z"`,

## changes-requested

- `specs/003-seed-distribution/contracts/seed-image-layout.md:79` — The timestamp contract says every inode carries `1970-01-01T00:00:00Z`, but `nix/seed-image.nix:42` only sets the image config `created` field. `dockerTools` writes layer tar entries with `--mtime=@$SOURCE_DATE_EPOCH`, and Nix's default `SOURCE_DATE_EPOCH` is `315532800` (`1980-01-01T12:00:00Z`), so this artifact will not satisfy the published contract unless the layer build explicitly sets/proves epoch-zero mtimes. Either make the implementation produce `1970-01-01T00:00:00Z` layer mtimes and keep the contract, or change the contract/acceptance text to the actual deterministic mtime the tar layer carries.
