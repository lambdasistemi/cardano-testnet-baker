## ready (rollback)

Round 1 of the determinism-bug fix. The reviewer's
`finalization.md` correctly blocked the PR on a real
determinism failure caught by T002's check on the
`normal` scenario (CI run
[25581132964](https://github.com/lambdasistemi/cardano-testnet-baker/actions/runs/25581132964/job/75100087121)).
Two independent builds produced different `layer.tar`
sha256s for `normal` (A=`2d3a6010…` vs B=`9abd6972…`),
while `local-fast` was deterministic.

The producer-side bug must live somewhere inside the
`normal` bake's seed payload. T002's check tells us
*that* the layer differs but not *which file* differs.
Round-1 of the fix lands a diagnostic enhancement before
the actual fix so the next CI run prints the offending
file path and per-file digests; the fix lands once the
bug is identified.

This is a new follow-up patch on top of T008
(`c6a8b3a`) rather than a T002 amend, deliberately —
amending T002 would force fresh review of T003 through
T008 (six already-approved patches). Carrying the fix
forward as a new vertical slice on top preserves those
approvals and matches the principle of small,
bisect-safe commits: this commit changes only the
diagnostic surface of one check and is provably a no-op
on green builds.

### What changed

`nix/checks.nix` (`determinismCheckScript`'s
`layer.tar` mismatch path):

- On `layer.tar` mismatch, extract both archives
  alongside their existing skopeo-derived
  `manifest.json`/config dumps.
- Build a sorted file list per side; `diff -u` the
  lists if they differ at all (catches missing-file
  drift).
- Walk the file list, `sha256sum` each side, print
  every entry where A and B disagree, with both
  digests. `MISSING` is logged when the file exists
  in A but not in B.
- Continues to `exit 1` after the diagnostic so the
  Build Gate stays red on real drift.

The existing per-layer sha logging, manifest dump, and
config dump on mismatch are unchanged.

### Why a NEW commit, not a T002 amend
