## ready

T002 from `specs/003-seed-distribution/tasks.md` —
`seed-image-determinism` check, wired into the Build Gate. Lands
on top of T001 (`03855bb`).

### What

- `nix/checks.nix`: takes new arguments `seedImage`, `scenariosDir`,
  `scenarioFiles` and exposes a new `seed-image-determinism`
  derivation. For each committed scenario, it:
  1. Builds the seed image twice as separate Nix derivations,
     using `overrideAttrs` to give each a distinct `name` (only
     the wrapping `runCommand`'s store path changes; the inner
     `streamLayeredImage` script + inputs are identical, so any
     byte-level drift is genuine non-determinism).
  2. Runs `skopeo inspect docker-archive:<archive>` on both,
     projects to the registry-side identity (`Architecture`,
     `Os`, `Created`, `Digest`, `Layers`,
     `LayersData[].{MIMEType,Digest,Size}`) and `diff -u`s the
     JSON. Any mismatch fails loudly.
  3. Asserts the *fixed* values explicitly:
     `Architecture == "amd64"`, `Os == "linux"`,
     `Created == "1970-01-01T00:00:00Z"`. Mutating the `created`
     timestamp in `nix/seed-image.nix` therefore fails the check
     (acceptance criterion).

- `flake.nix`: passes `seedImage`, `scenariosDir`, `scenarioFiles`
  through to `checks` so the determinism check can enumerate the
  same scenarios T001's `seedImage-<scenario>` packages
  enumerate.

- `.github/workflows/ci.yml`: adds
  `.#checks.x86_64-linux.seed-image-determinism` to the `Build
  Gate` job's `nix build` invocation, so every PR push has its
  determinism gate enforced before the rest of CI fans out.

- `justfile`: keeps `just build-gate` in sync with the CI workflow
  (the local mirror of the CI Build Gate now exercises the same
  set of checks).

### Validates / acceptance

- §II (determinism), spec FR-006, contract
  [`publish-pipeline.md §"Determinism check"`](../specs/003-seed-distribution/contracts/publish-pipeline.md),
  research §1, §8.
- Two consecutive `nix build .#checks.x86_64-linux.seed-image-determinism`
  invocations are idempotent — Nix returns the cached store path
  on the second call, and re-running the check after `nix store
  delete` rebuilds and produces the same store path.
- Mutating `created` in `nix/seed-image.nix` (e.g. to
  `"2026-05-08T00:00:00Z"` in a throwaway diff) fails the
  `jq -e '... .Created == "1970-01-01T00:00:00Z" ...'` assertion,
  per the acceptance.

  `jq 'del(.observation)' ...` step in
  `nix/seed-image.nix:extraCommands`) leaks the host-clock
  observation block into the layer; two builds at different host
  times produce different layer digests, so `diff -u
  <a.json> <b.json>` fails.

### Notes for the reviewer

- The `overrideAttrs (old: { name = "..."; })` trick is a
  deliberate way to materialise two store paths from a single
  logical derivation; without it, `mkSeedImage` with identical
  arguments returns the same store path twice and the diff is
  tautological. The trick changes only the wrapping
  `runCommand`'s `name` (and therefore the store-path hash); the
  inner `streamLayeredImage` script and its closure are byte-
  identical, so any drift in the produced tar is a real
  determinism bug.
- T001 introduced the new flake outputs and contracts; T002
  introduces the gate around them. Together they implement
  Phase 2 of the spec (foundational image + determinism
  enforcement).
- T003 (`compose/acceptance/run.sh` accepting an oci-archive
  URI) lands next on top of this patch.

## changes-requested

- `nix/checks.nix:18` — `base.overrideAttrs` is applied after `dockerTools.buildLayeredImage` has already captured the inner `streamLayeredImage` derivation, so `imageA` and `imageB` share the same layer-producing stream and only rebuild the outer archive wrapper. That makes the check too weak for T002: if `nix/seed-image.nix` reintroduces host-clock data in the customisation layer, both sides can still inspect the same already-built layer and compare equal. Construct two genuinely independent seed-image derivations while keeping the OCI name/tag/config stable, for example by threading a check-only derivation suffix/salt through `mkSeedImage` before the layer stream is created, then compare those two archives.
