Design observations (non-blocking, raise as tasks if relevant):

- Schema permits `{enabled: false, slotCount: 100}` — `slotCount` is
  effectively ignored when disabled. Acceptable but a stricter shape
  (forbidding `slotCount`/`profile` when `enabled = false`) would
  prevent silent misconfiguration. Mark as known limit if intentional.
- `slotCount` has `minimum: 1` but no upper bound — a future task
  should cap it once a sensible synthesis ceiling is known.
- `profile` is unconstrained beyond `minLength: 1`. Once profile
  identifiers stabilise, lift to an `enum` so misspellings fail at
  schema time.

Schema and tests are otherwise sound: root `additionalProperties: false`
already enforces declaration of `synthesis` as a property; the
conditional `allOf` correctly gates `slotCount` on `enabled = true`.
The TDD ordering workaround (combined patch with red evidence) is
fine; it resolves the legitimate tension between TDD and the
green-gate handoff.
